# Code of Conduct
**Structured Context Specification (SCS) Project**  
**Version:** 1.0  
**Last Updated:** 2025-11-17

---

## Our Core Principles

The SCS project is a collaborative effort focused on open, respectful, and productive teamwork between humans and AI agents. All contributors—whether maintainers, collaborators, or AI-assisted participants—are expected to contribute in a manner that supports:

- **Respect and professionalism**  
- **Open, constructive communication**  
- **Transparency and clarity**  
- **Kindness and patience**  
- **Inclusive collaboration**  
- **Intellectual honesty**  
- **Shared responsibility for improving the work**

This document outlines expected behavior and community standards.

---

## Expected Behavior

### 1. Be Respectful
Treat collaborators with respect, even when opinions differ.  
Critique ideas, not people.

### 2. Be Clear and Constructive
Provide feedback that is:
- Actionable  
- Specific  
- Courteous  
- Focused on improving the project  

### 3. Assume Positive Intent
Everyone is here to build something valuable.  
Misunderstandings happen—address them calmly.

### 4. Communicate Thoughtfully
Use Issues and Pull Requests to communicate clearly.  
Keep discussions on-topic and professional.

### 5. Work Transparently
All decisions should happen in:
- GitHub Issues  
- Pull Requests  
- Documented structured context files  

No decisions made privately.

### 6. Collaborate Inclusively
Support a welcoming environment for:
- Experienced contributors  
- New collaborators  
- AI-assisted contributions  
- Different viewpoints and backgrounds  

---

## Unacceptable Behavior

The following behaviors are not permitted:

- Harassment, discrimination, or personal attacks  
- Disrespectful or inflammatory language  
- Dismissing or belittling contributors  
- Off-topic trolling or disruptive behavior  
- Sharing private or sensitive information  
- Submitting misleading or intentionally harmful contributions  
- Using AI agents to generate abusive or harmful content  
- Pushing unreviewed changes directly to `main`  

---

## Responsibilities of Maintainters

Maintainers are expected to:

- Model respectful behavior  
- Uphold this Code of Conduct  
- Review contributions fairly and objectively  
- Address issues promptly and transparently  
- Enforce branching, PR, and workflow rules  
- Implement corrective actions when necessary  

Maintainers have the authority to remove, edit, or reject contributions that violate this Code of Conduct.

---

## Enforcement

If you observe behavior that violates this Code of Conduct:

1. Start with a polite note in the relevant Issue or Pull Request  
2. If unresolved, tag a maintainer for assistance  
3. For severe or persistent issues, maintainers may:
   - Remove comments  
   - Request behavioral changes  
   - Deny participation  
   - Block contributors  
   - Escalate to GitHub support if necessary  

All reports will be handled discreetly and respectfully.

---

## Using AI Tools Responsibly

AI agents are welcome contributors but must be used responsibly:

- Human contributors are accountable for AI-generated work  
- AI must not be used to override or disrupt human decisions  
- AI contributions must follow the same workflow and review process  
- AI interactions must respect the Code of Conduct  

---

## Acknowledgment

By participating in this project, you agree to uphold this Code of Conduct and support a positive, collaborative environment.

If you have questions, open an Issue and tag the maintainers.
